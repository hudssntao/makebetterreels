## Overview

Marketing landing page for **Kuantra**, built with Next.js (App Router), Tailwind CSS v4, and [shadcn/ui](https://ui.shadcn.com). The project is configured for light/dark themes, consistent typography with Geist fonts, and includes reusable UI primitives for future sections.

## Prerequisites

- Node.js ≥ 18
- [pnpm](https://pnpm.io/) ≥ 9 (project uses a `pnpm-lock.yaml`)

## Scripts

```bash
pnpm dev        # start local dev server
pnpm build      # produce a production build
pnpm start      # run the production build
pnpm lint       # run ESLint
pnpm lint:biome # run Biome lint pass
pnpm format     # format files in-place with Biome
pnpm format:check # verify formatting without writing changes
```

## Styling System

- **Tailwind CSS v4** with CSS variables generated by shadcn/ui
- **Fonts** are handled via `next/font`; see `src/app/layout.tsx` for Geist Sans and Geist Mono configuration
- **Components** live in `src/components/ui` (generated via `pnpm dlx shadcn@latest add <component>`)

## Theme Management

1. `src/components/theme-provider.tsx` wraps the app with [`next-themes`](https://github.com/pacocoursey/next-themes). The provider sets:
   - `attribute="class"` so Tailwind’s dark variants activate by toggling `class="dark"`
   - `defaultTheme="system"` while still allowing explicit light/dark modes
2. Global design tokens live in `src/app/globals.css`. The file defines the light palette on `:root` and overrides for dark mode within `.dark { ... }`. Update these CSS variables to change colors across the site.
3. To use theme-aware colors in components, rely on Tailwind design tokens (e.g. `bg-background`, `text-foreground`) or the shadcn-provided CSS variables.
4. Users toggle the theme through `ThemeToggle` in the header. The component consumes `useTheme()` and swaps between light/dark, respecting the system preference on first load.

## Font Configuration

- Fonts are declared once in `src/app/layout.tsx` using:
  ```ts
  const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"], display: "swap" });
  const geistMono = Geist_Mono({ variable: "--font-geist-mono", subsets: ["latin"], display: "swap" });
  ```
- The variables are attached to the `<body>` class list; Tailwind maps `font-sans` and `font-mono` to these through `globals.css`.
- To change the primary font, replace the `Geist` import with a different `next/font` helper and update the CSS variable names in both `layout.tsx` and `globals.css`.

## Adding Components

```bash
pnpm dlx shadcn@latest add card button badge
```

The generator uses `components.json` for configuration (aliases, theme file paths, etc.). Generated components are safe to customize; changes persist across subsequent additions.

## Project Structure

- `src/app/page.tsx` – landing page content and sections
- `src/components` – shared layout pieces (header, theme toggle/provider)
- `src/components/ui` – shadcn/ui primitives
- `src/lib/utils.ts` – helper utilities (e.g. `cn`)

## Deployment

1. Build locally with `pnpm build` to ensure the bundle compiles.
2. Deploy using your preferred platform (Vercel is recommended for Next.js). Ensure the environment uses pnpm or runs `corepack enable` before install.

## Biome & VS Code Setup

- Biome configuration lives in `biome.json`. It enables formatting, import organization, and the recommended lint rules.
- Run formatting manually with `pnpm format` or check formatting via `pnpm format:check`. Combine with `pnpm lint:biome` for Biome’s linter pass.
- VS Code is preconfigured under `.vscode/settings.json` to:
  - Enable `editor.formatOnSave`
  - Use the Biome extension (`biomejs.biome`) as the default formatter for TS/JS files
  - Trigger Biome import organization on save (explicit action so it can be toggled per file if needed)
- When opening the workspace, VS Code suggests installing the Biome extension through `.vscode/extensions.json`. Accept the recommendation for full integration.
